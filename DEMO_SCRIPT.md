# PixelForge AI — 演示视频脚本

> **时长**: 3 分钟 | **风格**: 产品演示 + 实战案例 | **配音**: 男声，沉稳专业，节奏明快

---

## 角色设定

### 主角：小李
- **身份**: 独立游戏开发者，正在用 Godot 引擎制作一款像素风 RPG 游戏
- **痛点**: 美术零基础，网上找的素材风格不统一；用通用 AI 生图工具产出的图没有 `.meta`/`.tres` 文件，无法直接导入引擎
- **目标**: 在 10 分钟内完成一套风格统一的游戏图标（药水、宝剑、盾牌、背包），并导入 Godot 使用

---

## 分镜脚本

### 第一幕：痛点共鸣 (0:00 - 0:25)

```
画面: 分屏对比
  左侧 — 小李在 Godot 里拖入从网上找的素材，风格割裂（写实药水 + 卡通宝剑）
  右侧 — 小李打开 PixelForge AI，输入一行文字

旁白:
  "独立游戏开发者最怕什么？不是写代码，是找素材。
  网上东拼西凑的风格割裂，AI 生图工具产出的图片没有引擎文件。
  从创意到可用的游戏资产，中间隔着一条看不见的鸿沟。

  直到他发现了 PixelForge AI。"
```

### 第二幕：核心演示 — 精灵生成 (0:25 - 1:05)

```
画面: 浏览器打开 http://127.0.0.1:7860
  展示 5 个 Tab: 精灵生成 | 瓦片生成 | UI生成 | 特效生成 | 动画生成

  小李在"精灵生成"Tab 操作:

  风格:   [像素风 (Pixel Art)]  ← 下拉选择
  模板:   [RPG 药水]            ← 下拉选择
  描述:   health potion, red liquid, glass bottle, cork stopper, glowing
  负面:   (留空)
  尺寸:   [图标 256x256]
  Seed:   [42]

  点击 [生成] 按钮
  进度条:  构建提示词... → 调用 AI 生成中... → 完成

  右侧出现一张像素风红色药水图标

旁白:
  "打开 PixelForge AI，五个功能标签页一目了然。
  选择像素风，选 RPG 药水模板，描述你要的药水——红色液体、玻璃瓶、软木塞。

  点击生成，等待约 15 秒——一张风格纯正的像素风药水图标就出现了。
  但这只是开始。"
```

### 第三幕：批量生成 + 引擎导出 (1:05 - 1:45)

```
画面:
  小李展开"高级设置"

  批量数量: [▮▮▮▮] 4

  输入素材列表（每行一个）:
    health potion, red liquid, glowing
    iron sword, sharp blade, golden hilt
    wooden shield, metal rim, round
    leather backpack, brown, adventure gear

  点击 [批量生成]

  进度条: 生成第 1/4 个... → 生成第 2/4 个... → ...

  Gallery 显示 4 张风格统一的像素风图标

  小李选择导出引擎: [Godot]
  点击 [导出资源包]
  下载 ZIP 文件

  画面切到文件管理器:
    assets/
    ├── 1x/
    │   ├── ...potions.png
    │   ├── ...sword.png
    │   ├── ...shield.png
    │   └── ...backpack.png
    ├── 2x/
    └── GODOT_README.txt

旁白:
  "更强大的是批量生成。一次输入四个素材描述，选择同一种风格模板，
  PixelForge 确保它们视觉风格完全统一。

  导出时选择 Godot——点击导出，得到一个完整的资源包：
  一倍图、二倍图、还有 .tscn 场景文件和 .import 配置。
  这就是 PixelForge 的核心价值：不只是生图，是交付引擎就绪的资产。"
```

### 第四幕：动画生成 (1:45 - 2:10)

```
画面: 小李切到"动画生成"Tab

  角色描述: knight warrior, silver armor, blue cape
  风格:      [像素风 (Pixel Art)]
  动画类型:  [行走动画 (Walk)]
  尺寸:      [精灵 512x512]

  点击 [生成动画帧]

  Gallery 显示 4 帧行走动画序列:
    帧1: 左脚前迈
    帧2: 中间步态
    帧3: 右脚前迈
    帧4: 收回站姿

  小李把这 4 帧导入 Godot 的 SpriteFrames，角色在场景里走动起来。

旁白:
  "游戏不只是静态图标。切到动画生成，描述你的角色，
  选择行走动画——四帧序列一次生成。
  每一帧之间通过种子递增保持动作连贯，直接导入 Godot 就能动起来。

  从一张图到一个会动的角色，只需要一句描述。"
```

### 第五幕：瓦片集 + 配色方案 (2:10 - 2:35)

```
画面: 小李切到"瓦片生成"Tab

  风格:      [像素风 (Pixel Art)]

  输入:  grassland terrain tile, green grass, seamless

  点击 [生成]

  画面展示生成的草皮瓦片、边缘瓦片和角落瓦片

  叠字: "支持 Auto-Tile 规则瓦片 — 中心 + 4边 + 4角"

旁白:
  "PixelForge 还支持地形瓦片生成。
  草地、地牢、森林、科幻——四种预置主题，
  甚至支持 Auto-Tile 规则瓦片：中心、四边、四角一次生成，
  导入 Tiled 或 Unity Rule Tile 直接可用。"
```
